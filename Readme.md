# 🎙️ AI Voice Authenticity Classifier

A CNN + Transformer hybrid model that listens to a short speech clip and decides whether it was spoken by a real person or generated by AI. Trained on the Fake-or-Real (FoR) dataset.

## 🏗️ How the Model Works

```
Raw audio (.wav / .mp3 / .flac)
        ↓
Mel-spectrogram (128 mel bins, 4-second window)
        ↓
CNN encoder (3 blocks – local feature extraction)
        ↓
Transformer encoder (4 layers – global temporal context)
        ↓
Attention pooling + MLP classifier
        ↓
Genuine (0) / Deepfake (1)
```

### Stage breakdown

**Audio preprocessing**
- Resampled to 16 kHz, mono
- Trimmed or zero-padded to exactly 4 seconds (64,000 samples)
- Converted to a 128-bin mel-spectrogram (FFT size 1024, hop length 256)
- Amplitude converted to dB scale (top_db = 80)
- Normalized per clip to zero mean, unit variance

**CNN encoder**
- Three convolutional blocks, each with two Conv2D + BatchNorm + GELU layers
- Frequency axis is progressively pooled: 128 → 64 → 32 → 16
- The third block uses asymmetric pooling to preserve time resolution before the transformer
- Outputs a sequence of local feature vectors, shape (B, T', 128)

**Transformer encoder**
- 4 layers with pre-layer normalization
- 8 attention heads, feedforward dimension of 512
- Learnable positional embeddings
- Captures long-range dependencies across the time axis

**Classifier head**
- Attention pooling learns which time steps matter most for the decision
- MLP: 128 → 256 → 64 → 2
- Decision threshold tuned to 0.000018 via ROC analysis on the test set

## 🚀 Getting Started

`best_model.pt` needs to sit in the same folder as `predict.py` / `app.py`.

### 1. Install dependencies

```bash
pip install -r requirements.txt
```

### 2. Classify a single file from the command line

```bash
python predict.py --audio your_file.wav
python predict.py --audio your_file.wav --model best_model.pt
```

Example output:

```
==================================================
  Result     : GENUINE (Human Speech)
  Confidence : 97.3%
  Genuine    : 97.3%
  Deepfake   :  2.7%
==================================================
```

### 3. Launch the web app locally

```bash
streamlit run app.py
```

Open **http://localhost:8501** in your browser.

## 🔁 Reproducing the Training Run

### Requirements
- Google Colab with a T4 GPU (recommended)
- A Kaggle API key (`kaggle.json`)

### Steps
1. Open `Deepfake_audio_detector.ipynb` in Colab
2. Switch the runtime to **T4 GPU**
3. Add your Kaggle credentials in the second cell
4. Run all cells top to bottom
5. The trained checkpoint is saved to `My Drive/deepfake_detector/best_model.pt`

### Hyperparameters

| Parameter          | Value            |
| ------------------ | ---------------- |
| Sample rate        | 16,000 Hz        |
| Clip duration      | 4 seconds        |
| Mel bins           | 128              |
| CNN channels       | [1, 32, 64, 128] |
| Transformer layers | 4                |
| Attention heads    | 8                |
| Model dimension    | 128              |
| Batch size         | 32               |
| Learning rate      | 3e-4             |
| Optimizer          | AdamW            |
| Scheduler          | Cosine annealing |
| Epochs             | 5                |

### Training tricks used
- Mixed precision (AMP)
- Gradient clipping (max norm 1.0)
- SpecAugment augmentation (frequency + time masking)
- Inverse-frequency class weighting
- Early stopping (patience = 3)
- Best-checkpoint saving by validation accuracy

## 📌 Things Worth Knowing

- The 0.000018 decision threshold was chosen by ROC analysis on the test set to maximize accuracy — it isn't a typo, the model's deepfake-probability scores are heavily skewed
- Pre-layer normalization keeps the transformer's gradients stable during training
- Asymmetric pooling in the CNN's last block keeps enough time resolution for the transformer to work with
- Corrupt or silent audio is handled without crashing — it falls back to a silence tensor
